Agent Movement with Graph search and Reinforced Learning

Agent Movement

I want to learn how to make an agent - some automated object, such as a car or NPC - move from point A to point B. It should be able to dodge collisions and reroute on the fly.

Problem

video games contain many repetitive tasks and they take to long to complete. Repetitve tasks include fetching items back and forth from NPCs, item seeking and gathering, or defeating X number of opponents.

Solution

An agent capable of moving from point A to B; more specifically: complete fetching, gathering, or defeating tasks.

Its movement and pathfinding doesn't have to be perfect or most optimal just yet. We'll explore and document enhancements in future articles.
